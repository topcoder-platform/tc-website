Event Engine

The Event Engine is an event driven service that manages delivery of notifications (hereafter known as events). The EventEngine works with three plugable components: registry handler, alert handler and event handler. The registry handler is used by the EventEngine to retrieve information related to the registered alert and event handlers. An alert handler is responsible for polling and creating event to be delivered by the EventEngine. An event handler is responsible for delivery of the events.

The Event Engine comes bundled with three handlers: a configuration file registry handler, a Microsoft SQL Server alert handler and a TCS document generation and email component event handler.

As an example, the default handlers can be used to create a message to be sent via email. The message would be stored in the SQL Server database for some delivery time. The alert handler would poll the database for the message and create an event to be delivered by the EventEngine. The EventEngine would then hand this event off to the event handler for formatting and sending the document via email.
