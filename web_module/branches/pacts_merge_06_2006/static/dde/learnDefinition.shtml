What is a component?

This seemingly simple question is not answered as easily as one might think. The technical definition of a component is an encapsulated set of logic accessed through strictly defined interfaces. The commercial definition is more complicated.

TopCoder Software defines a commercial component as a definitive set of business or technical logic that performs a discrete function for customers. From a business perspective, components are not useful standing by themselves. They become useful when combined with other components to form component assemblies. Component assemblies are combinations of components that together perform a specific business function.

As an example, TopCoder Software has in its software catalog a Configuration Manager component. The Configuration Manager component manages the contents and caching of properties files for software applications. When bundled together with a few other standard TopCoder Software Components namely event engine, email engine and document generator they together form the Notification Component Assembly. This becomes a very useful tool for scheduling and sending form emails.
